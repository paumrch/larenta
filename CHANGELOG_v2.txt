1. CORRECCION DEL FILTRO DE SITUACIONES (Bug fix)
--------------------------------------------------

Problema:
- Si el usuario no seleccionaba ninguna situacion, TODAS las deducciones
  pasaban el filtro.
- 58 deducciones tenian situaciones: [] (vacio), pasaban siempre.
- Las respuestas de alquiler y ganancias se recogian pero no se usaban.

Solucion:

  Logica (Wizard.tsx):
  - Antes:  if (answers.situaciones.length > 0 && d.situaciones.length > 0)
  - Despues: if (d.situaciones.length > 0)

  Datos (39 deducciones reclasificadas):
  - 21 donativos
  - 3 vehiculo electrico (coche_electrico)
  - 4 zonas rurales (vive_en_pueblo)
  - 3 vivienda propia (tiene_vivienda_propia)
  - 2 empleados hogar (empleados_hogar)
  - 2 discapacidad
  - 4 inversion/autonomos (invierte / autonomo)
  - 19 restantes dejadas vacias (aplicacion general)

3. NUEVO PASO DE EDAD EN EL WIZARD
------------------------------------

Nuevo paso 3 (de 7 totales): pregunta la edad del contribuyente.

Pasos del Wizard (antes → despues):
  0: CCAA → CCAA
  1: Laboral → Laboral
  2: Situaciones → Situaciones
  3: Alquiler → Edad (NUEVO)
  4: Ganancias → Alquiler
  5: Economicos → Ganancias
  6: — → Economicos

Detalle:
- Input numerico con autoFocus
- Texto de ayuda: "Hay deducciones especificas para jovenes (<36) y mayores (>65)"
- Opcional: el usuario puede avanzar sin rellenar

Logica de filtrado:
  if (d.edad_maxima != null && edad > d.edad_maxima) return false;
  if (d.edad_minima != null && edad < d.edad_minima) return false;

Datos de edad:
- 2 campos nuevos en 375 deducciones: edad_minima, edad_maxima
- 27 deducciones con valores reales (jovenes <36, mayores >65/70/75)
- 38 con edades en nombre pero aplican a dependientes (no filtrables)
